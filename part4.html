📖 Topic Overview — Greedy Set Cover Approximation (CLO-5)

The Problem: You have a universe U of n elements and a collection of m subsets S₁, S₂, …, Sₘ where each Sᵢ ⊆ U. Find the smallest sub-collection of sets whose union equals U — cover every element using as few sets as possible.

Why is this hard? Set Cover is NP-hard — no polynomial-time exact algorithm is known (and none is believed to exist). Brute-force over all 2m subset collections is only feasible for small m. For large instances we need an approximation algorithm: one that runs in polynomial time and guarantees a solution within a provable factor of optimal.

The Greedy Algorithm:

C ← ∅ // selected cover (starts empty) U' ← U // remaining uncovered elements while U' ≠ ∅: S* ← argmax |Sᵢ ∩ U'| // pick set covering most uncovered C ← C ∪ {S*} U' ← U' \ S* return C

Approximation Ratio — H(d): Let d = max|Sᵢ| and H(k) = 1 + 1/2 + 1/3 + … + 1/k (harmonic number, ≈ ln k + 0.577). The greedy algorithm always produces a cover of size ≤ H(d) · OPT. For d = 10, H(10) ≈ 2.93, meaning greedy uses at most about 3× as many sets as optimal.

Proof sketch (CLRS Theorem 35.4): Let OPT = k*. After the greedy algorithm selects t sets, let nₜ = |uncovered elements|. Before step 1: n₀ = n elements remain; at least one set in OPT covers ≥ n₀/k* of them, so greedy picks a set of size ≥ n₀/k*, leaving n₁ ≤ n₀(1 − 1/k*). By induction after t steps: nₜ ≤ n·(1 − 1/k*)ᵗ ≤ n·e^(−t/k*). Setting nₜ < 1 (i.e., everything covered) requires t > k*·ln n. So greedy terminates in ≤ k*·ln n steps = OPT · H(n) ≤ OPT · H(d).

OPT Computation (Brute Force): For small instances we can verify the greedy gap exactly — enumerate all 2m − 1 non-empty subsets of sets, track the minimum number of sets that covers all of U. This runs in O(2m · n) time and is only feasible for m ≤ 20 or so, but it lets us see the exact ratio greedy/OPT on every example.

Inapproximability: Dinur and Steurer (2014) proved that no polynomial-time algorithm can achieve ratio better than (1 − ε) · ln n for any ε > 0 unless P = NP. Greedy achieves ln n ≈ H(n), so it is essentially optimal among polynomial algorithms.

Time Complexity: Each of the ≤ n greedy rounds scans all m sets and computes intersection sizes: O(n · m · n) naively, or O(n · m) with a coverage counter. Total: O(n² · m) naively or O(n · m) with careful bookkeeping.

What is Set Cover?

Given a universe U of n elements and a collection S of subsets, find the minimum number of subsets whose union equals U.

Formally: find C ⊆ S such that ⋃C = U and |C| is minimised.
Decision version: can we cover U with ≤ k sets? — NP-complete (Karp, 1972).

Examples: placing sensors to monitor all locations, selecting database indexes to cover all queries, choosing test cases to cover all code paths.

NP-Hardness & Why Greedy?

Set Cover is NP-hard — no known polynomial-time exact algorithm exists unless P = NP.

Exact approach: try all 2^|S| subsets → exponential, infeasible.
Greedy approach: polynomial time O(n · |S|), provably near-optimal.
Key insight: greedy gives an H(n)-approximation — the solution is at most ln(n)+1 times the optimal. For n=100, that's ≤ 5.6× OPT. Often much better in practice.
Greedy Algorithm
GREEDY-SET-COVER(U, S): covered ← ∅ cover ← [] while covered ≠ U: for each s in S: gain(s) ← |s \ covered| s* = argmax{ gain(s) } cover.append(s*) covered ← covered ∪ s* return cover // ≤ H(n)·|OPT|

At each step, pick the set covering the most currently uncovered elements. Repeat until all elements are covered.

Approximation Ratio H(n)

The greedy algorithm produces a cover of size at most H(n) · |OPT|.

H(n) = 1 + 1/2 + 1/3 + … + 1/n ≈ ln(n) + 0.577
H(1)=1, H(2)=1.5, H(4)≈2.08, H(8)≈2.72, H(100)≈5.19
Why H(n)? If OPT needs k sets, the best uncovered set covers ≥ 1/k fraction of remaining elements. Each iteration "pays off" at most H(n)/OPT steps total — like the coupon collector problem.
Proof of H(n) Bound

The elegant charging argument:

Assign cost 1/gain(s*) to each element newly covered by s*.
Total cost = |cover| (one cost-1 set covers 1st new element, etc.).
Each element e pays at most 1/1 + 1/2 + … = H(n_e) where n_e = elements uncovered when e was first covered.
Since OPT covers all elements and must pay for each: |OPT| ≥ n/max_set_size. Combining: |greedy| ≤ H(n) · |OPT|. □
Inapproximability
Hardness result (Dinur & Steurer, 2014): Unless P = NP, no polynomial algorithm can approximate Set Cover within (1−ε)·ln(n) for any ε > 0.
Greedy achieves H(n) ≈ ln(n)+1 — this is essentially optimal among poly-time algorithms!

This means greedy is not just a practical heuristic — it is theoretically optimal in terms of approximation ratio for a polynomial-time algorithm.

Weighted Set Cover (each set has a cost): same greedy idea, pick set with best cost-per-new-element ratio. Same H(n) approximation guarantee holds.
Real-World Applications
Network monitoring: place minimum sensors to observe all traffic flows.
Facility location: open minimum warehouses to serve all customers.
Bioinformatics: minimum set of DNA probes covering all target genes.
Compiler optimisation: minimum test suite covering all code branches (covering arrays).
Advertisement: minimum ad slots covering the entire target audience.
Set Cover ↔ Vertex Cover: SC generalises vertex cover; Hitting Set is the dual problem.
Complexity & Variants
Greedy time: O(n · |S| · iterations) — each iteration scans all sets.
With a max-heap: O(n · |S| · log|S|) — faster for large inputs.
Space: O(n + |S|)
k-Set Cover: each set has ≤ k elements → (1 − 1/e) approximation via LP rounding.
Online Set Cover: elements revealed one at a time — greedy still works but ratio degrades to O(log n · log|S|).
Partial Cover: cover at least p fraction of U — greedy adapted with same guarantee.
