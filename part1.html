1. Why Fast Fourier Transform?

Multiplying two degree-n polynomials naively takes O(n²) time — for each coefficient of A you multiply by every coefficient of B. This becomes prohibitively slow for large polynomials used in signal processing, image compression (JPEG), large integer arithmetic, and convolution in deep learning.

The FFT reduces this to O(n log n) by converting the problem from coefficient representation to point-value representation, where multiplication is trivially O(n), then converting back.

2. Polynomial Representations

A polynomial of degree n−1 can be represented in two equivalent ways:

Coefficient Representation

A(x) = a₀ + a₁x + a₂x² + … + aₙ₋₁xⁿ⁻¹

Stored as array [a₀, a₁, …, aₙ₋₁]. Easy to add. Slow to multiply: O(n²).

Point-Value Representation

A = {(x₀, y₀), (x₁, y₁), …, (xₙ₋₁, yₙ₋₁)} where yᵢ = A(xᵢ)

n+1 points uniquely determine a degree-n polynomial. Multiply pointwise: O(n).

Key insight: FFT is just a very fast way to convert between these two representations using special evaluation points.

3. DFT and Roots of Unity

The Discrete Fourier Transform evaluates a polynomial at n-th roots of unity: the complex numbers ωₙᵏ = e^(2πik/n) for k = 0, 1, …, n−1.

Why roots of unity?

  • They lie evenly spaced on the unit circle in the complex plane
  • Cancellation property: ω₂ₙᵏ⁺ⁿ = −ω₂ₙᵏ — the second half is the negative of the first half
  • Halving property: squaring the 2n-th roots gives the n-th roots, enabling recursion
  • These two properties are the mathematical engine of FFT

The primitive n-th root of unity is ωₙ = e^(2πi/n). For n=8: ω₈ = e^(iπ/4) = (√2/2) + (√2/2)i

4. The Butterfly Network

The FFT computes DFT using a divide-and-conquer structure visualized as a butterfly network. Each "butterfly" is a 2-input, 2-output operation:

// Basic butterfly at twiddle factor ω:
a' = a + ω·b
b' = a − ω·b

For n inputs, there are log₂(n) levels, each with n/2 butterflies. Total: (n/2)log₂(n) butterfly operations. Each butterfly is O(1), so total is O(n log n).

Bit-Reversal Permutation

Before the butterfly network, inputs are reordered by reversing the bits of their indices. For n=8: index 3 (binary 011) → reversed 110 = 6. This puts the even/odd recursion in natural order.

5. The FFT Algorithm (Cooley-Tukey)

FFT(a[0..n-1]): // n must be power of 2 if n == 1: return a ω_n = e^(2πi/n) // primitive n-th root of unity ω = 1 a_even = [a[0], a[2], a[4], …] // even-indexed coefficients a_odd = [a[1], a[3], a[5], …] // odd-indexed coefficients y_even = FFT(a_even) // recurse on n/2 y_odd = FFT(a_odd) // recurse on n/2 for k = 0 to n/2 − 1: y[k] = y_even[k] + ω · y_odd[k] // butterfly upper y[k + n/2] = y_even[k] − ω · y_odd[k] // butterfly lower ω = ω · ω_n // advance twiddle factor return y

Recurrence: T(n) = 2T(n/2) + Θ(n). By Master Theorem (Case 2): T(n) = Θ(n log n).

6. Inverse FFT (Interpolation)

The inverse DFT converts point-values back to coefficients. Remarkably, it's almost identical to FFT:

IFFT = FFT with conjugate roots + divide by n

Replace ωₙ with ωₙ⁻¹ = e^(−2πi/n) and divide every output by n.

This works because the DFT matrix is unitary (up to scaling): DFT⁻¹ = (1/n)·DFT*

7. The 3-Step Polynomial Multiplication Pipeline

Step 1 — Evaluate (FFT)

Compute FFT(A) and FFT(B) — evaluate both polynomials at 2n roots of unity. Cost: O(n log n) each.

Step 2 — Pointwise Multiply

C[k] = A[k] × B[k] for each k. Cost: O(n).

Step 3 — Interpolate (IFFT)

Apply IFFT to C to recover coefficient representation. Cost: O(n log n).

Total: O(n log n) vs O(n²) naive — a dramatic speedup for large n.

8. Complexity Comparison

Concrete Numbers

  • n = 1,000: Naive = 1,000,000 ops  |  FFT ≈ 10,000 ops  → 100× faster
  • n = 1,000,000: Naive = 10¹² ops  |  FFT ≈ 20,000,000 ops  → 50,000× faster
  • n = 10⁹: Naive = 10¹⁸ ops  |  FFT ≈ 3×10¹⁰ ops  → 33,000,000× faster

9. Fully Worked Example

A(x) = 1 + 2x,   B(x) = 3 + 4x   →   C(x) = A·B = ?

Naive: C = (1)(3) + (1·4 + 2·3)x + (2)(4)x² = 3 + 10x + 8x²


Via FFT (n=4 points):

  • Pad: A = [1,2,0,0], B = [3,4,0,0]
  • Roots: ω₄⁰=1, ω₄¹=i, ω₄²=−1, ω₄³=−i
  • FFT(A) = [3, 1−2i, −1, 1+2i]
  • FFT(B) = [7, 3−4i, −1, 3+4i]
  • Pointwise: [21, (1−2i)(3−4i), 1, (1+2i)(3+4i)] = [21, −5−10i, 1, −5+10i]
  • IFFT → [3, 10, 8, 0] → C(x) = 3 + 10x + 8x² ✓
