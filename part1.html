📖 Topic Overview — Fast Fourier Transform & Polynomial Multiplication (CLO-2)

What is this topic? This tool covers the Fast Fourier Transform (FFT) algorithm and how it solves the polynomial multiplication problem in O(n log n) time — a dramatic improvement over the naïve O(n²) approach. The FFT is one of the most important algorithms in computer science and is used in signal processing, image compression, cryptography, and more.

The Problem: Given two polynomials A(x) = a₀ + a₁x + a₂x² + … + aₙ₋₁xⁿ⁻¹ and B(x) = b₀ + b₁x + … + bₙ₋₁xⁿ⁻¹, compute their product C(x) = A(x) · B(x). The result has degree deg(A) + deg(B) and up to 2n−1 coefficients.

Why is the naïve approach too slow? The straightforward method multiplies every coefficient of A by every coefficient of B — that's n × n = n² multiplications. For small polynomials this is fine. But in practice, polynomials can have thousands of terms. For n = 10,000 terms, naïve needs 100 million multiplications. For n = 1,000,000 (common in signal processing), that's a trillion operations — completely infeasible in real time.

Key Insight — Evaluate, Multiply, Interpolate:
Instead of multiplying coefficients directly, we can:
1. Evaluate both polynomials at n cleverly chosen points → O(n log n)
2. Multiply the values at each point: C[k] = A[k] × B[k] → O(n)
3. Interpolate back to get the coefficients of C → O(n log n)
Total: O(n log n). The trick is choosing the right n points so steps 1 and 3 are fast.

Why roots of unity? The n-th roots of unity are the complex numbers ωₙᵏ = e^(2πik/n) for k = 0, 1, …, n−1. They have two algebraic properties that make the recursion collapse:
— Cancellation: ω^(k + n/2) = −ωᵏ, so the second half of outputs mirrors the first with a sign flip — no extra computation.
— Halving: (ωₙᵏ)² = ω_{n/2}^k, so squaring maps n-th roots to (n/2)-th roots — the subproblem shrinks by half each level.

The Even-Odd Split (Cooley-Tukey, 1965):
Split A by even and odd index: A(x) = A_even(x²) + x · A_odd(x²)
where A_even = [a₀, a₂, a₄, …] and A_odd = [a₁, a₃, a₅, …]

To evaluate A at all n roots of unity, we only need to evaluate A_even and A_odd at n/2 roots — the problem halves! Applying this recursively gives the recurrence:
T(n) = 2T(n/2) + Θ(n)
By the Master Theorem (Case 2, a=2, b=2, log₂2=1): T(n) = Θ(n log n)

The Butterfly Operation: At each level of the recursion, pairs of values are combined using the formula:
   t = ω · y_odd[k]    (twiddle factor)
   y[k] = y_even[k] + t    (upper output)
   y[k + n/2] = y_even[k] − t    (lower output, free from cancellation!)
With log₂(n) levels and n/2 butterflies per level, total butterflies = (n/2) · log₂(n) = O(n log n).

Inverse FFT: To recover coefficients from point-values, we use the Inverse FFT — identical to FFT but with the conjugate twiddle ω⁻¹ = e^(−2πi/n) and a final division by n. The DFT matrix Dₙ is unitary: Dₙ⁻¹ = (1/n)D̄ₙ. This closes the evaluate-multiply-interpolate pipeline perfectly.

Worked Example: A = [1, 2, 3, 4], B = [1, 1, 1, 1]
Naïve: 4 × 4 = 16 multiplications  |  FFT: pad to n=8, 3 levels, ~40 operations
Result: C = [1, 3, 6, 10, 10, 9, 7, 4] → C(x) = 1 + 3x + 6x² + 10x³ + 10x⁴ + 9x⁵ + 7x⁶ + 4x⁷ ✓
At scale (n=1,000): naïve = 1,000,000 ops vs FFT ≈ 10,000 ops → 100× speedup

Why FFT?

Naïve polynomial multiplication: for each coefficient of A, multiply by every coefficient of B → O(n²).

FFT exploits roots of unity to evaluate, multiply pointwise, then interpolate back → O(n log n).

3-Step Pipeline:
1. FFT(A) and FFT(B) — evaluate at roots of unity
2. Pointwise: C[k] = A[k] × B[k] — O(n)
3. IFFT(C) — interpolate back to coefficients
Roots of Unity & Butterfly

Evaluate at the n-th roots of unity: ωₙᵏ = e^(2πik/n) for k = 0…n−1.

Cancellation: ω^(k+n/2) = −ωᵏ → second half mirrors first half.
Halving: (ωₙᵏ)² = ω_{n/2}^k → recursion halves the problem.

Each butterfly: t = ω·b; a' = a+t; b' = a−t
log₂(n) levels × n/2 butterflies = O(n log n).

Cooley-Tukey Algorithm
FFT(a[0..n-1]): if n == 1: return a a_even = a[0,2,4,…] // divide a_odd = a[1,3,5,…] y_e = FFT(a_even) y_o = FFT(a_odd) ω_n = e^(2πi/n); ω = 1 for k = 0 to n/2−1: t = ω · y_o[k] // twiddle y[k] = y_e[k] + t // butterfly ↑ y[k+n/2] = y_e[k] − t // butterfly ↓ ω = ω · ω_n

Recurrence: T(n) = 2T(n/2) + Θ(n) → Θ(n log n) by Master Theorem Case 2.

Divide & Conquer Analysis

The key algebraic identity that makes FFT work:

A(x) = A_even(x²) + x · A_odd(x²)
At ωₙᵏ: A(ωₙᵏ) = A_even(ω_{n/2}ᵏ) + ωₙᵏ · A_odd(ω_{n/2}ᵏ)
At ωₙ^(k+n/2): A(ωₙ^(k+n/2)) = A_even(ω_{n/2}ᵏ) − ωₙᵏ · A_odd(ω_{n/2}ᵏ)

Why this is powerful: the second half of the DFT outputs is just the first half with a sign flip — no extra work! Each level re-uses the same sub-DFT results twice via the cancellation property ω^(k+n/2) = −ω^k.

Master Theorem: T(n) = 2T(n/2) + Θ(n)
a=2, b=2, f(n)=Θ(n), log_b(a)=1 → Case 2: T(n) = Θ(n log n)
Inverse FFT & Interpolation

The DFT matrix Dₙ has entries Dₙ[j,k] = ω^(jk). Its inverse is:

D⁻¹ = (1/n) · D̄ (conjugate, scaled)
IFFT = conjugate twiddle ω̄ = e^(−2πi/n), then divide by n

So IFFT is identical code to FFT — just negate the exponent sign and scale. This is why the 3-step pipeline closes perfectly:

Uniqueness guarantee: n points uniquely determine a polynomial of degree ≤ n−1. Using n-th roots of unity as evaluation points + DFT = fast interpolation. IFFT recovers the unique degree-(n−1) polynomial passing through all n point-values.
Applications & Variants

FFT is one of the most impactful algorithms ever discovered (Cooley & Tukey, 1965):

Signal Processing: audio compression (MP3), spectrum analysis — FFT runs in real-time on hardware chips
Image: JPEG uses DCT (Discrete Cosine Transform) — a variant of FFT
Large Integer ×: multiply 1,000-digit numbers in O(n log n log log n)
Neural Networks: convolution in CNNs via FFT — same butterfly math
NTT (Number Theoretic Transform): FFT over a finite field ℤ_p instead of ℂ — exact integer arithmetic, used in cryptography (lattice-based schemes, polynomial multiplication in CRYSTALS-Kyber)
Complexity lower bound: Ω(n log n) for DFT — FFT is asymptotically optimal!
Numerical Precision & NTT

FFT uses complex floating-point arithmetic — rounding errors accumulate across butterfly levels.

Problem: For large inputs or integer coefficients, float errors can corrupt results (e.g. 2.9999… instead of 3).
Fix 1: Round to nearest integer at the end — works when coefficients are known integers.
Fix 2: Use NTT — FFT over ℤ_p (a prime field) instead of ℂ.
Number Theoretic Transform (NTT): replace ωₙ = e^(2πi/n) with a primitive n-th root of unity mod p. All arithmetic is exact integers — no float errors. Used in cryptography (CRYSTALS-Kyber, ML-KEM post-quantum standard) and competitive programming.
Common Mistakes & Edge Cases
Wrong padding size: n must be ≥ deg(A) + deg(B) + 1, not just max(len(A), len(B)). Under-padding causes circular convolution (aliasing).
Off-by-one: A polynomial of degree d has d+1 coefficients. Always count coefficients, not degree.
Circular vs. linear convolution: DFT computes circular convolution by default. Zero-padding to length ≥ len(A)+len(B)−1 converts it to linear convolution = polynomial multiplication.

In this tool n = next_pow2(len(A)+len(B)−1) — always correct for polynomial multiplication.

Why not evaluate at arbitrary points?

Any n+1 points uniquely determine a degree-n polynomial. The trick is choosing the n-th roots of unity — they have algebraic structure (cancellation + halving properties) that lets the evaluation decompose recursively.
A(x) = A_even(x²) + x · A_odd(x²)
Splitting by even/odd index halves the problem at each level. With log₂(n) levels and n/2 butterflies per level: O(n log n).

Inverse FFT

IFFT is identical to FFT but uses ω⁻¹ = e^(−2πi/n) as the twiddle factor and divides all outputs by n. The DFT matrix is unitary (up to 1/√n scaling).
Worked Example:
A=[1,2,3,4], B=[1,1,1,1]. Pad to n=8. FFT(A) and FFT(B) evaluate at 8 roots of unity, multiply pointwise (8 ops), IFFT → C=[1,3,6,10,10,9,7,4] ✓
