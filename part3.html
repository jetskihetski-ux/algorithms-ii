What is a Convex Hull?

The convex hull of a point set S is the smallest convex polygon that contains all points in S — the "rubber band stretched around the points."

A set is convex if for any two points A, B ∈ S, the entire line segment AB ⊆ S.
Key property: hull vertices are always a subset of the original points — interior points are never hull vertices.

Applications: collision detection, path planning, GIS, pattern recognition, alpha shapes, gift packing.

Cross Product Orientation Test

The orientation of three ordered points O→A→B determines everything:

cross(O, A, B) = (A.x−O.x)·(B.y−O.y) − (A.y−O.y)·(B.x−O.x)
> 0 : left turn (counterclockwise) — B is left of OA ✓
= 0 : collinear
< 0 : right turn (clockwise) — B is right of OA ✗

This O(1) test is the core primitive for both Graham Scan and Jarvis March. No trigonometry needed — pure arithmetic.

Graham Scan — O(n log n)
GRAHAM_SCAN(pts): pivot = lowest_y(pts) // anchor sort pts by polar angle from pivot stack = [pivot, pts[0]] for p in remaining pts: while cross(s[-2],s[-1],p) ≤ 0: stack.pop() // right turn → off hull stack.push(p) return stack // CCW hull

Stack invariant: at all times, the stack forms a valid CCW convex chain. Each point is pushed/popped at most once → O(n) scan after O(n log n) sort.

Jarvis March — O(nh)
JARVIS(pts): start = leftmost(pts) hull = []; cur = start repeat: hull.append(cur) cand = pts[0] for p in pts: if cross(cur, cand, p) > 0: cand = p // more CCW cur = cand until cur == start return hull

Each of the h hull vertices requires one O(n) scan. Total: O(nh). Best when h ≪ n (few hull vertices). Worst case O(n²) when all points on hull.

Comparison & Lower Bound
Ω(n log n) lower bound: sorting reduces to convex hull — any hull algorithm implies a sort, so no deterministic algorithm can beat O(n log n).
Graham: O(n log n) always — best for large h
Jarvis: O(nh) — best when h is small (h = O(log n))
Chan's (1996): O(n log h) — optimal output-sensitive algorithm combining both ideas

In practice, Graham Scan is preferred due to its predictable performance and cache-friendly stack operations.

Why does Graham Scan work?

The stack maintains a CCW convex chain. When we process a new point p in sorted polar-angle order, any right turn on the stack top means the previous point is "inside" the chain formed by the new point — so it is provably not a hull vertex. The while-loop removes all such violations, restoring convexity invariant before pushing p.

Invariant: after processing sorted[i], the stack contains the convex hull of {pivot, sorted[0], …, sorted[i]}.

Why sort by polar angle?

Sorting ensures we process points in angular order around the pivot. This means each "right turn" detection exactly identifies a non-hull point — we never need to revisit earlier points after they are popped.

⚠ Common Misconception

"All points on the boundary are hull vertices." — FALSE. Collinear boundary points are NOT hull vertices (they lie on an edge, not a vertex). Graham Scan pops them (cross product = 0 → ≤ 0 → pop).

Worked Example:
Points: (10,10) (90,10) (90,90) (10,90) (50,50)
Pivot: (10,10) — lowest y, leftmost
Sorted by angle: (90,10) 0° → (90,90) 45° → (10,90) 90°
Stack trace: [(10,10),(90,10)] → push (90,90) → push (10,90)
Hull: {(10,10),(90,10),(90,90),(10,90)} — interior (50,50) excluded ✓

Jarvis Correctness

At each step, we pick the point most counterclockwise from the current hull vertex. This is equivalent to finding the point p that minimizes the polar angle from the current direction. No other point can lie to the left of the line from current to p, so p must be the next hull vertex.
