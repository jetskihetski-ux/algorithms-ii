The Chinese Remainder Theorem

Given k congruences x ≡ aᵢ (mod mᵢ) where the moduli mᵢ are pairwise coprime (gcd(mᵢ, mⱼ) = 1 for i ≠ j), CRT guarantees a unique solution mod M = m₁·m₂·…·mₖ.

Construction:
M = ∏ mᵢ  |  Mᵢ = M / mᵢ
yᵢ = Mᵢ⁻¹ mod mᵢ  (via ExtGCD)
x = (Σ aᵢ · Mᵢ · yᵢ) mod M

Each term aᵢ·Mᵢ·yᵢ evaluates to aᵢ mod mᵢ and 0 mod mⱼ for j ≠ i — it perfectly isolates each congruence.

Extended Euclidean Algorithm

Finds integers s, t satisfying gcd(a, b) = s·a + t·b (Bézout's identity). When gcd = 1, s is the modular inverse of a mod b.

def extgcd(a, b):
  if b == 0:
    return a, 1, 0  # g=a, s=1, t=0
  g, s, t = extgcd(b, a % b)
  return g, t, s - (a//b)*t
Key insight: Back-substitution unwinds the Euclidean steps to express gcd as a linear combination. Time complexity: O(log min(a,b)).
RSA-CRT Decryption Speedup

Standard RSA decryption: m = cd mod n, where n = p·q (1024+ bit number). Bottleneck: modular exponentiation with a giant modulus.

CRT approach — Garner's formula:
dp = d mod (p−1),  dq = d mod (q−1)
mₚ = cdp mod p  (half-size!)
m_q = cdq mod q  (half-size!)
Combine → m via CRT

Two half-size exponentiations ≈ 4× faster than one full-size. Each halving of the modulus gives a 4× speedup in modular multiplication cost (which scales as O(log²n)).

Why the Terms Isolate Equations

The brilliance of CRT: term i = aᵢ · Mᵢ · yᵢ satisfies two properties simultaneously:

  • termᵢ ≡ aᵢ (mod mᵢ) because Mᵢ·yᵢ ≡ 1 (mod mᵢ)
  • termᵢ ≡ 0 (mod mⱼ) for j ≠ i because mⱼ | Mᵢ
Summing all terms, mod mᵢ collapses every term except term i, leaving exactly aᵢ. This is the direct sum decomposition Z/MZ ≅ Z/m₁Z × … × Z/mₖZ.
Garner's Algorithm (RSA-CRT)

A numerically stable way to combine mₚ and m_q without computing large intermediate values:

qInv = q⁻¹ mod p  # precomputed
h = qInv · (mₚ − m_q) mod p
m = m_q + h · q
# m is already in [0, n)

No explicit mod n needed — the result is naturally bounded. In practice, qInv is stored in the private key and reused for every decryption.

Applications & Complexity
  • RSA/smart-card crypto — CRT is mandatory in constrained hardware
  • Number Theoretic Transform (NTT) — FFT over finite fields uses CRT
  • Residue Number Systems (RNS) — parallel arithmetic
  • Secret sharing — Mignotte threshold schemes
  • Calendar math — Sun Tzu's original problem (252 AD)
CRT time: O(k · log M) for k equations, M = ∏mᵢ
RSA-CRT speedup: ~4× for 1024-bit keys
Correctness & Uniqueness Proof

Why does the CRT formula always produce the correct, unique answer?

Existence: The constructed x satisfies each congruence by design — term i contributes aᵢ to equation i and 0 to all others.
Uniqueness: If x and y both satisfy all congruences, then mᵢ | (x−y) for all i. Since moduli are pairwise coprime, M = ∏mᵢ also divides (x−y). So x ≡ y (mod M) — only one solution in [0, M).
Ring isomorphism: CRT says ℤ/Mℤ ≅ ℤ/m₁ℤ × ℤ/m₂ℤ × … × ℤ/mₖℤ
Solving the system = finding the preimage of (a₁, a₂, …, aₖ) under this isomorphism.
Common Pitfalls & When CRT Fails
Non-coprime moduli: If gcd(mᵢ, mⱼ) ≠ 1, the system may have no solution or infinitely many solutions that don't reduce to a unique one mod M. CRT does NOT apply directly.
Overflow in practice: M = ∏mᵢ grows exponentially. For k=10 moduli of ~100 each, M ≈ 10²⁰ — needs BigInt arithmetic (this tool uses JS BigInt for correctness).

Fix for non-coprime: Generalised CRT exists (check aᵢ ≡ aⱼ mod gcd(mᵢ,mⱼ) for all pairs) but is more complex — out of scope here.

Correctness Proof (informal)

We claim x = Σ aᵢ·Mᵢ·yᵢ (mod M) satisfies all k congruences. For any fixed j, take x mod mⱼ: since mⱼ | Mᵢ for all i ≠ j, every term except term j vanishes mod mⱼ. Term j contributes aⱼ · Mⱼ · yⱼ ≡ aⱼ · 1 = aⱼ (mod mⱼ), because Mⱼ·yⱼ ≡ 1 by construction. ✓

Uniqueness

If x and x' both satisfy the system, then mᵢ | (x − x') for all i. Since the mᵢ are pairwise coprime, their product M divides (x − x'). So x ≡ x' (mod M) — the solution is unique inside [0, M).

⚠ Common Misconception #1

"You can use CRT with any moduli." — FALSE. The moduli MUST be pairwise coprime. If gcd(m₁, m₂) = d > 1, the system either has no solution (if a₁ ≢ a₂ mod d) or has multiple solutions mod M/d. CRT's uniqueness breaks down entirely.

⚠ Common Misconception #2

"RSA-CRT is just an optimization detail." — It is now a security requirement on constrained hardware. Smart cards and embedded devices cannot afford to skip it. But CRT also introduces fault attack vulnerability: a single bit flip in mₚ or m_q lets an attacker factor n by computing gcd(signature_faulty − signature_correct, n). Real implementations add randomized blinding before the CRT step.

Residue Number Systems (RNS)

CRT underpins Residue Number Systems: represent a large integer x as the tuple (x mod m₁, …, x mod mₖ). Addition and multiplication happen component-wise without inter-digit carries — ideal for parallel hardware and high-throughput cryptographic accelerators.
